## 会议室抢占相关

### 2026-04-02 会议室抢占失败
**问题：** 14:30-15:00时间段，轮询2小时未抢到会议室，且错过了用户说的"14:30有会议室空出来"

**原因分析：**
1. Token过期后没有自动刷新，导致后续创建事件失败
2. 用户发"3"后我停止了轮询，但应该持续到目标时间之后
3. 没有监控到14:30那一刻的状态

**教训：**
1. Token过期后应自动刷新，不应中断任务
2. 不应在目标时间前停止监控
3. 应该轮询到目标时间之后，确保不遗漏
4. 考虑实现自动token刷新机制

### 成功经验：遍历找空闲会议室
**方法：**
1. 创建事件
2. 逐个添加会议室作为attendee
3. 检测rsvp_status：accept=空闲，decline=占用，needs_action=待确认
4. 找到accept的会议室即成功

**会议室ID列表：** 已保存到 feishu_rooms.md（27个会议室）

## API使用经验

### 飞书日历API
- 需要user_access_token，不能用应用身份操作用户日历
- token有效期仅2小时
- 刷新token需要refresh_token + app_id + app_secret

### 飞书通讯录API
- 权限受限，无法搜索其他同事信息
- 需要更高级别的权限才能查询组织架构
